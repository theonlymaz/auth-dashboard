ðŸš« .gitignore FILE
===============================================

Add these to EVERY Python project:

venv/              # Virtual environment (too big)
.env               # Secrets (NEVER share!)
__pycache__/       # Python cache (auto-generated)
*.pyc              # Compiled Python files
.DS_Store          # Mac hidden file (annoying)
*.log              # Log files
.vscode/           # VS Code settings (optional)

ðŸ—„ï¸ SUPABASE SETUP
===============================================

1. Get credentials from supabase.com:
   - Project URL
   - anon/public key

2. Add to .env file:
   SUPABASE_URL=https://xxx.supabase.co
   SUPABASE_KEY=eyJhbGc...

3. Connect in Python:
   from supabase import create_client
   import os
   
   supabase = create_client(
       os.getenv("SUPABASE_URL"),
       os.getenv("SUPABASE_KEY")
   )

ðŸ› TROUBLESHOOTING

   â†’ Make sure virtual environment is activated
   â†’ Should see (venv) in terminal

Module not found?
   â†’ Install it: pip install module-name
   â†’ Check you're in right environment

Port already in use?
   â†’ Kill process: lsof -ti:8501 | xargs kill -9

Can't connect to Supabase?

   â†’ Make sure load_dotenv() is called
   â†’ Verify credentials in Supabase dashboard

ðŸ’¡ IMPORTANT CONCEPTS
===============================================

Virtual Environment:
   â†’ Isolated space for each project's packages
   â†’ Like separate toolboxes for each project
   â†’ Prevents package conflicts between projects

Local vs Deployed:
   â†’ Local: Runs on your computer only
   â†’ Deployed: Live on internet, anyone can access

Environment Variables:
   â†’ Sensitive info (API keys, passwords)
   â†’ Never put directly in code
   â†’ Always use .env file
   â†’ Never commit .env to Git!
